# Audit Evolution 现场 Demo 脚本

## 90 秒版本

开场：

> 大部分 Agent 不是不会回答，而是不知道自己刚刚发生了什么。长任务之后，它会混淆旧状态、新状态、权威文件、失败原因和下一步。Audit Evolution 给 Agent 装了一个飞行记录仪，让它每跑一轮都留下可审计、可恢复、可进化的记录。

亮结果：

- Longju SACP Operator: `93.0/100`，S 级，全球 #1。
- Jobs: `88.8/100`，A 级，全球 #4。
- 重点不是一次高分，而是失败后能自动定位短板并生成下一轮最小补丁。

展示黑盒：

```text
上下文 99k/131k。
已读 5 个文件。
不知道哪个分数是当前权威。
不确定任务是否完成。
还想继续扫描。
```

切换到 Audit Evolution：

```text
Evidence Pack:
先列证据和缺失证据。

Snapshot:
区分 trusted_state 和 uncertain_state。

Evolution Card:
指出本轮最该修哪一个能力维度。

Memory Ledger:
只沉淀带证据、可过期、可检索的记忆。

Patch Proposal:
只给一个最小补丁。
```

结尾：

> 我们不是让 Agent 只完成一次任务，而是让它从每次运行中进化。用户不用学协议，只要回一句：进化。

## 如果现场网络不好

直接打开：

```text
index.html
```

然后按这三屏讲：

1. 成绩截图：证明真实跑过。
2. 黑盒痛点：证明问题真实存在。
3. 闭环输出：证明这个 skill 不是总结器，而是进化器。

## 如果有人问“我的 Agent 没崩溃还需要吗”

回答：

> 需要。Audit Evolution 不是崩溃修复器，而是每次任务后的飞行记录仪。成功、失败、跑分、用户纠错、上下文过载、skill 修改，都应该变成下一轮更好的启动状态。

## 如果有人问“会不会乱改我的项目”

回答：

> 默认不会。它先审计和提出补丁，真正修改、发布、上传、安装、跑官方 benchmark，都要人类批准。
